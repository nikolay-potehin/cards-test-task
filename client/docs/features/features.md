## Conventions
Each `docs/features/<feature_name>/<feature_name>.md` file contains:
- **User story**: the feature's intent from the user's perspective.
- **Actions flow**: an ordered list of user actions and expected outcomes, sufficient to write integration tests. No implementation details (no file paths, class names, constants, or internal mechanics).
